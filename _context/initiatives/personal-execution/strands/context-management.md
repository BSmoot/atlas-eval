# Work Strand: Context Management

**ID:** WS-005
**Status:** Active
**Created:** 2026-01-26
**Owner:** Ben Smoot

---

## Purpose

Establish context management for the business and its people alongside context management for the business and its AI. Create the shared understanding layer that enables both humans and AI to work effectively - individually and together.

---

## Core Principle

Any newly onboarded agent or human should have access to the same foundational context. The difference is in how they consume it, not what they have access to.

### 7. Traceability Without Bloat

Notes, decisions, specs - traceable and documented without overwhelming core context.

| Layer | Content | Access Pattern |
|-------|---------|----------------|
| Core | Canonical docs, current state | Always loaded |
| Reference | Decisions, specs, rationale | Retrieved on demand |
| Archive | Historical notes, superseded docs | Search when needed |

**Principle:** Keep signal high in core context. Link to detail, don't embed it.

**Deliverable:** Context Architecture (core/reference/archive)
